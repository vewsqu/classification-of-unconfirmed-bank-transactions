# Ядро антифрод-системы: Data Fusion Contest 2026 (Задача "Страж")

[cite_start]Репозиторий содержит пайплайн для решения задачи бинарной классификации неподтвержденных (мошеннических) банковских операций на основе истории транзакций клиентов[cite: 2, 4, 5].

## Архитектура решения

Решение представляет собой монолитный пайплайн, реализованный в `main.py`, который покрывает все этапы: от извлечения сырых данных до обучения модели и финального ансамблирования (блендинга).

### 1. Высокопроизводительная обработка (Polars)
Для обработки более 200 миллионов записей используется библиотека `polars` в режиме ленивых вычислений (`LazyFrame`). Это позволяет:
* [cite_start]Эффективно считать исторические агрегаты (средние траты, максимальные суммы, количество транзакций) [cite: 252-258].
* [cite_start]Использовать оконные функции для расчета времени с последней транзакции (`sec_since_last`) и изменения часового пояса [cite: 208-219].
* [cite_start]Генерировать статистику по конкретным MCC-кодам и устройствам для каждого пользователя [cite: 259-260].

### 2. Векторизация поведения (Word2Vec)
Ключевая фишка решения — применение NLP-подходов к табличным данным.
* [cite_start]**Токенизация:** Каждая транзакция превращается в текстовый токен вида `MCC_Сумма_Время` (например, объединение кода категории продавца, логарифмированного бина суммы и времени суток)[cite: 132].
* [cite_start]**Эмбеддинги:** История транзакций клиента рассматривается как "предложение", на котором обучается модель `Word2Vec` от `gensim` (размер вектора 32) [cite: 135-137].
* [cite_start]**Профилирование:** Создается "вектор профиля" пользователя (среднее его транзакций) и вычисляется косинусное сходство (`w2v_cosine_similarity`) между текущей операцией и типичным поведением клиента [cite: 147-148, 174].

### 3. Обучение модели (CatBoost)
В качестве основного классификатора используется `CatBoostClassifier`, обучаемый на GPU:
* [cite_start]Из-за сильного дисбаланса классов (мошеннических операций очень мало) применяется взвешивание `scale_pos_weight=5.0`[cite: 16, 324].
* [cite_start]Обучение идет на 3500 итерациях с ранней остановкой (early stopping) по метрике PR-AUC [cite: 316, 320-322].
* [cite_start]Категориальные признаки (MCC, устройства, типы ОС) передаются в модель напрямую без предварительного One-Hot кодирования [cite: 287-291].

### 4. Ансамблирование (Blending)
В конец скрипта встроен модуль блендинга. [cite_start]Он позволяет смешать вероятности лучшей модели CatBoost с предсказаниями других моделей (например, LightGBM) с помощью взвешенного среднего (например, 60% / 40%) для повышения итоговой метрики и стабильности на приватном лидерборде [cite: 355-357].
