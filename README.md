# Ядро антифрод-системы: Data Fusion Contest 2026 (Задача "Страж")

Репозиторий содержит пайплайн для решения задачи бинарной классификации неподтвержденных (мошеннических) банковских операций на основе истории транзакций клиентов.

## Архитектура решения

Решение представляет собой монолитный пайплайн, реализованный в `main.py`, который покрывает все этапы: от извлечения сырых данных до обучения модели и финального ансамблирования (блендинга).

### 1. Высокопроизводительная обработка (Polars)
Для обработки более 200 миллионов записей используется библиотека `polars` в режиме ленивых вычислений (`LazyFrame`). Это позволяет:
* Эффективно считать исторические агрегаты (средние траты, максимальные суммы, количество транзакций).
* Использовать оконные функции для расчета времени с последней транзакции (`sec_since_last`) и изменения часового пояса.
* Генерировать статистику по конкретным MCC-кодам и устройствам для каждого пользователя.

### 2. Векторизация поведения (Word2Vec)
Ключевая фишка решения — применение NLP-подходов к табличным данным.
* **Токенизация:** Каждая транзакция превращается в текстовый токен вида `MCC_Сумма_Время` (например, объединение кода категории продавца, логарифмированного бина суммы и времени суток).
* **Эмбеддинги:** История транзакций клиента рассматривается как "предложение", на котором обучается модель `Word2Vec` от `gensim` (размер вектора 32).
* **Профилирование:** Создается "вектор профиля" пользователя (среднее его транзакций) и вычисляется косинусное сходство (`w2v_cosine_similarity`) между текущей операцией и типичным поведением клиента.

### 3. Обучение модели (CatBoost)
В качестве основного классификатора используется `CatBoostClassifier`, обучаемый на GPU:
* Из-за сильного дисбаланса классов (мошеннических операций очень мало) применяется взвешивание `scale_pos_weight=5.0`.
* Обучение идет на 3500 итерациях с ранней остановкой (early stopping) по метрике PR-AUC.
* Категориальные признаки (MCC, устройства, типы ОС) передаются в модель напрямую без предварительного One-Hot кодирования.

### 4. Ансамблирование (Blending)
В конец скрипта встроен модуль блендинга. [cite_start]Он позволяет смешать вероятности лучшей модели CatBoost с предсказаниями других моделей (например, LightGBM) с помощью взвешенного среднего (например, 60% / 40%) для повышения итоговой метрики и стабильности на приватном лидерборде.
